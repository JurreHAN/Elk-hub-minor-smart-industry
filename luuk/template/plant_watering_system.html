Short Project Overview

Smart Plant Watering System

A device that takes care of your plant, so you don't have to.

Hardware Setup

  • ESP32 Microcontroller
  • Custom PCB Design
  • Light Dependent Resistor (LDR)
  • Relay Module
  • Soil Moisture Sensor
  • DHT11 Temp & Humidity

Key Features

  • Real-time data
  • Automated watering logic
  • Compact and reliable design
  • Remote App control via Blynk
  • Web dashboard for monitoring

Overview

The goal of this IoT project was to design an automated plant watering system that monitors soil moisture, light intensity, temperature, and humidity. The system uses various sensors connected to an ESP32 microcontroller to collect data and control a relay for watering the plant when needed.

Hardware Design

In the first workshops I learned how to use a breadboard to connect the sensors and relay to the ESP32 (see Figure 1). This was essential for prototyping and testing the circuit before finalizing the design. I learned a lot about how you can build IoT devices. I did once work with microcontrollers before, but I only soldered components and the code was already made. So this was my first time programming a microcontroller and it really showed me how easily you can create a simple smart devices like this automatic water planting system.

After testing the components on the breadboard and writing all the code, I had another workshop. This was about designing a custom PCB using Eagle (see Figure 2). During this workshop I created a compact PCB layout to neatly arrange all components and connections. It was very satisfying to see how all the wires from the breadboard prototype could be organized into a single small PCB. At first I was kind of overwhelmed by the fact that I had to design the entire PCB myself, because I do not understand much about electronics yet. But after following the workshop and asking questions to the teacher, I managed to create a decent PCB layout (and it worked 😁).

The PCB layout included footprints for the ESP32, sensors, relay, and connectors for power and data. After finalizing the design, my school placed an order. When the PCB arrived, I soldered all components during a soldering workshop (see Figure 3). I did not really learn anything new during the soldering workshop, because I already had experience with soldering. But it was nice to solder my own designed PCB and help others with their soldering.

Software Development

The ESP32 was programmed using PlatformIO in VSCode. A large part of the code was given during the workshops, but I had to modify and expand it to implement the specific functionality of the plant watering system. This way we didn't have to search for the right libraries and write all code from scratch, which saved a lot of time. The code lets the ESP32 read data from the soil moisture sensor, LDR, and DHT11 sensor. When the soil moisture drops below 30%, the relay is activated to water the plant for a few seconds. I implemented a simple logic to avoid overwatering by adding a delay between watering cycles. When demonstrating the project I lowered this delay to 1 minute to show the functionality quickly. I also integrated Blynk to enable remote monitoring and control via a mobile app. The app displays real-time sensor data and allows manual activation of the watering system. The data collected by sensors is even sent to a database with a web dashboard for live monitoring.
